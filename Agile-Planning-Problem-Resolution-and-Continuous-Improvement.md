# ‚≠ê 1. **What is Adaptive Planning?**

Adaptive planning means **planning iteratively and updating the plan based on feedback**, not making a fixed, rigid plan at the beginning.

### Example

In a food delivery app project, if new competitor features appear, the next sprint plan can adapt and include similar features.

# ‚≠ê 3. **Progressive Elaboration**

# ‚≠ê 6. **Timeboxing**

A fixed amount of time allocated for an activity.

Agile uses:

* 2-week sprints
* 15-minute standups
* 2-hour retrospectives

### Example

A spike to research a new API is timeboxed to 8 hours.

# ‚≠ê 7. **Estimation**

In Agile, estimation is:

* Relative (not exact)
* Collaboration-based
* Story points or t-shirt sizes

---

# ‚≠ê 8. **Ideal Time**

Ideal time = How long a task would take **if there were no interruptions**.

But because real work includes distractions ‚Üí Agile prefers **story points**.

# ‚≠ê 9. **User Stories**

User stories capture requirements from the user's perspective.

### Example

As a traveler, I want to book a hotel so I can plan my trip easily.

# ‚≠ê 12. **Story Points and T-Shirt Sizing**

---

# ‚≠ê 13. **Wideband Delphi**

Anonymous expert estimation technique.

Steps:

1. Experts estimate individually
2. Moderator collects estimates
3. Group discusses differences
4. Repeat until convergence

---

# ‚≠ê 14. **Planning Poker**

A gamified version of estimation.

Steps:

1. Product Owner explains user story
2. Each team member chooses a Fibonacci card
3. All reveal at same time
4. Discuss differences
5. Re-estimate until agreement
